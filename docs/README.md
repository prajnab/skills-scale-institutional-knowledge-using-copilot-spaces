# OctoAcme Project Management Docs

This folder contains OctoAcme's project management process documents and acts as the single entry point for onboarding, execution, and continuous improvement.

Summary
-------
OctoAcme uses a structured, iterative approach to deliver customer value safely and predictably. Key principles:
- Customer-first value delivery
- Iterative, measurable progress (small increments)
- Clear roles and ownership (PM, PdM, developers, QA, stakeholders)
- Data-informed decisions and metrics
- Psychological safety, learning, and continuous improvement

Lifecycle (high level)
1. Initiation — validate problem, confirm stakeholders, create a Project One-pager.
2. Planning — scope, backlog, estimates, release plan, and risk register.
3. Execution & Tracking — deliver in small increments, track progress and metrics, manage blockers.
4. Release & Deployment — verify pre-release checks, deploy via pipelines, run post-deploy checks.
5. Retrospective & Continuous Improvement — capture learnings, convert to action items.

Where to start
--------------
- Read the Project Management Overview for a concise orientation.
- Complete the Project One-pager in Project Initiation for any new initiative.
- Follow the Planning document to turn the One-pager into a backlog and release plan.
- Use Execution & Tracking during delivery; update the Risk Register as needed.
- Use Release & Deployment when preparing to ship to production.
- Run a Retrospective after milestones/releases to create actionable improvements.
